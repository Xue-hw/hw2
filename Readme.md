基于范式哈夫曼编码的通用无损压缩系统本系统是数据结构课程设计作业二的最终成果，旨在实现一个稳定、高效且通用的无损数据压缩工具。系统核心基于范式哈夫曼编码（Canonical Huffman Coding）算法，并针对大规模中英文语料（如《红楼梦》、《莎士比亚全集》）进行了深度优化。🚀 核心特性通用字节流处理：不再局限于纯文本。通过底层的 rb 模式读取，系统可无损处理图片、PDF及二进制文件 。范式化码表优化：采用范式哈夫曼逻辑，压缩包头部仅需存储“字符-码长”映射（Lengths Map），无需存储复杂的树拓扑结构，极大地精简了元数据开销 。字符级中文优化：针对中文 UTF-8 编码特性，通过字符级统计模型，将中文文本的压缩率从 26% 提升至 45% 以上 。流式解压架构：引入 Python 生成器（Generator）与分块读取（64KB Chunking）机制，将内存占用控制在极低水平，并实现 10 倍于传统逻辑的解压速度 [12, 12]。健壮性保障：内置对单一字符文件、超大路径双引号处理等极端边缘情况的防御逻辑 。🛠️ 技术实现细节1. 算法逻辑建树阶段：利用 heapq 最小堆构建最优二叉树，确保加权路径长度最短 。范式化阶段：通过双重排序（码长升序 + 字典序升序）重新分配编码，保证同长度编码数值连续 。持久化：使用 pickle 模块对精简后的码长表进行序列化存储 。2. 补齐机制 (Padding)系统在位流头部存储 1 字节的补齐位计数，确保解压端能精确剔除字节对齐时产生的多余比特位 。📂 文件说明huffman.py: 核心逻辑库。包含 HuffmanCoder 类，处理建树、范式编码分配、位流转换等 。test.py: 自动化测试与展示脚本。提供终端可视化界面、高频字符统计、性能指标计算（Saving/Ratio）及 MD5 级一致性校验 [12, 12]。📖 运行指南环境要求Python 3.10 或更高版本。外部依赖：仅需 requests（用于自动下载测试语料集）。启动测试直接运行 test.py 即可进入交互模式：Bashpython test.py
